## 为什么写博客

我一直觉得，**写作是最好的思考方式**。当你有想法在脑子里打转时，它们往往是模糊的、跳跃的。但一旦开始写下来，你就会被迫面对那些含糊不清的地方，把它们理清楚。

这个博客就是为了这个目的而生的——我想有一个地方，能把平时工作中积累的经验、学到的技术、生活中的观察，有条理地记录下来。

## 技术选型

这次搭建博客，我选用了 [Astro](https://astro.build)，主要基于几个考量：

- **内容优先**：Astro 天生为内容型网站设计，Markdown 支持开箱即用
- **零 JS 默认，按需加载**：博客大部分页面是纯静态的，不需要 JS，但当你需要交互时又能用 React/Vue/Svelte
- **性能极致**：构建出的静态页面在 Lighthouse 评分上几乎是满分
- **扩展性**：未来想加什么功能（评论、搜索、数据可视化），都能轻松接入

样式方面用了 **Tailwind CSS** + **Typography** 插件，配色选择深色主题，希望提供舒适的阅读体验。

## 文章格式

所有文章都使用 Markdown 编写，放在 `src/data/blog/` 目录下。每一篇文章的开头需要用 frontmatter 来描述元信息：

```markdown
---
title: "标题"
description: "简短描述"
date: 2026-06-04
tags: ["标签1", "标签2"]
---

正文内容...
```

`draft: true` 可以标记草稿，草稿文章不会出现在任何列表和 RSS 中。

## 接下来的计划

博客的基本框架已经搭好了，但还有很多想做的事：

1. **评论系统**——已支持登录后评论
2. **全文搜索**——用 Pagefind 实现客户端搜索
3. **图片优化**——自动化图片压缩和响应式
4. **SEO 优化**——sitemap、OG 标签等
5. **更多内容**——这是最重要的，持续输出

希望这个博客能成为一个记录成长的地方。如果你也在考虑搭建自己的博客，不妨现在就动手——**最好的启动时间就是现在**。
